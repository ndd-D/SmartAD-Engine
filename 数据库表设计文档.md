根据 API 文档和项目需求，我设计了以下 6 张核心表。

# 智投引擎数据库设计文档 v1.0

## 通用规范
- **字符集**：`utf8mb4`
- **排序规则**：`utf8mb4_general_ci`
- **引擎**：`InnoDB`
- **时间字段**：统一使用 `datetime` 类型

---

## 1. 用户表（`smartad_user`）

| 字段名          | 类型     | 长度 | 允许空 | 默认值                      | 说明                      |
| --------------- | -------- | ---- | ------ | --------------------------- | ------------------------- |
| id              | bigint   | 20   | NO     | 自增                        | 主键ID                    |
| username        | varchar  | 50   | NO     | -                           | 用户名（唯一索引）        |
| password        | varchar  | 100  | NO     | -                           | BCrypt加密密码            |
| nickname        | varchar  | 50   | NO     | -                           | 昵称                      |
| role            | varchar  | 20   | NO     | 'admin'                     | 角色（第一阶段只有admin） |
| status          | tinyint  | 1    | NO     | 1                           | 状态：1正常 0禁用         |
| last_login_time | datetime | -    | YES    | NULL                        | 最后登录时间              |
| create_time     | datetime | -    | NO     | CURRENT_TIMESTAMP           | 创建时间                  |
| update_time     | datetime | -    | NO     | CURRENT_TIMESTAMP ON UPDATE | 更新时间                  |

**索引**：
- 主键：`id`
- 唯一索引：`username`

**初始数据**：
```sql
INSERT INTO smartad_user (username, password, nickname, role) VALUES 
('admin', '$2a$10$XgVYqQqQqQqQqQqQqQqQqOqQqQqQqQqQqQqQqQqQqQqQqQqQq', '运营管理员', 'admin');
-- 密码为：123456 (BCrypt加密)
```

## 2. 指令表（`smartad_command`）

| 字段名        | 类型     | 长度 | 允许空 | 默认值                      | 说明                                               |
| ------------- | -------- | ---- | ------ | --------------------------- | -------------------------------------------------- |
| id            | bigint   | 20   | NO     | 自增                        | 自增主键（用于序号生成）                           |
| command_id    | varchar  | 30   | NO     | -                           | 指令ID（格式：CMD+日期+序号，唯一索引）            |
| user_id       | bigint   | 20   | NO     | -                           | 下发用户ID，关联smartad_user.id                    |
| command_text  | varchar  | 500  | NO     | -                           | 原始指令内容                                       |
| status        | varchar  | 20   | NO     | '待AI处理'                  | 状态：待AI处理/处理中/AI提问中/已生成策略/执行失败 |
| ai_question   | varchar  | 500  | YES    | NULL                        | AI提问内容（状态为AI提问中时填写）                 |
| fail_reason   | varchar  | 500  | YES    | NULL                        | 失败原因                                           |
| strategy_id   | varchar  | 30   | YES    | NULL                        | 关联的策略ID（生成后填写）                         |
| reply_content | varchar  | 500  | YES    | NULL                        | 前端补充回复内容                                   |
| reply_time    | datetime | -    | YES    | NULL                        | 回复时间                                           |
| create_time   | datetime | -    | NO     | CURRENT_TIMESTAMP           | 创建时间                                           |
| update_time   | datetime | -    | NO     | CURRENT_TIMESTAMP ON UPDATE | 更新时间                                           |

**索引**：
- 主键：`id`
- 唯一索引：`command_id`
- 普通索引：`status`（用于AI轮询查询）
- 普通索引：`user_id`

| 字段名          | 类型     | 长度 | 允许空 | 默认值                      | 说明                                                         |

## 第一阶段初始化SQL脚本

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS smartad_engine DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

USE smartad_engine;

-- 用户表
CREATE TABLE smartad_user (
    id BIGINT(20) NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    username VARCHAR(50) NOT NULL COMMENT '用户名',
    password VARCHAR(100) NOT NULL COMMENT 'BCrypt加密密码',
    nickname VARCHAR(50) NOT NULL COMMENT '昵称',
    role VARCHAR(20) NOT NULL DEFAULT 'admin' COMMENT '角色',
    status TINYINT(1) NOT NULL DEFAULT 1 COMMENT '状态：1正常 0禁用',
    last_login_time DATETIME NULL COMMENT '最后登录时间',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (id),
    UNIQUE KEY uk_username (username)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';

-- 指令表
CREATE TABLE smartad_command (
    id BIGINT(20) NOT NULL AUTO_INCREMENT COMMENT '自增主键',
    command_id VARCHAR(30) NOT NULL COMMENT '指令ID（格式：CMD+日期+序号）',
    user_id BIGINT(20) NOT NULL COMMENT '下发用户ID',
    command_text VARCHAR(500) NOT NULL COMMENT '原始指令内容',
    status VARCHAR(20) NOT NULL DEFAULT '待AI处理' COMMENT '状态：待AI处理/处理中/AI提问中/已生成策略/执行失败',
    ai_question VARCHAR(500) NULL COMMENT 'AI提问内容',
    fail_reason VARCHAR(500) NULL COMMENT '失败原因',
    strategy_id VARCHAR(30) NULL COMMENT '关联的策略ID',
    reply_content VARCHAR(500) NULL COMMENT '前端补充回复内容',
    reply_time DATETIME NULL COMMENT '回复时间',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (id),
    UNIQUE KEY uk_command_id (command_id),
    KEY idx_status (status),
    KEY idx_user_id (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='指令表';

-- 策略表
CREATE TABLE smartad_strategy (
    id BIGINT(20) NOT NULL AUTO_INCREMENT COMMENT '自增主键',
    strategy_id VARCHAR(30) NOT NULL COMMENT '策略ID（格式：STR+日期+序号）',
    command_id VARCHAR(30) NULL COMMENT '关联指令ID',
    strategy_name VARCHAR(100) NOT NULL COMMENT '策略名称',
    channels VARCHAR(100) NOT NULL COMMENT '渠道，JSON数组格式',
    budget_day INT(11) NOT NULL COMMENT '日预算（元）',
    crowd_tag VARCHAR(50) NOT NULL COMMENT '人群标签',
    bid_price DECIMAL(10,2) NOT NULL COMMENT '出价（元）',
    run_time VARCHAR(30) NOT NULL COMMENT '投放时段',
    product_type VARCHAR(30) NULL COMMENT '产品类型',
    risk_level VARCHAR(10) NOT NULL DEFAULT '普通' COMMENT '风险等级：普通/高风险',
    status VARCHAR(20) NOT NULL DEFAULT '待确认' COMMENT '状态：待确认/待人工确认/投放中/已暂停/已下线/已拒绝/超时失效',
    confirm_user_id BIGINT(20) NULL COMMENT '确认人ID',
    confirm_time DATETIME NULL COMMENT '确认时间',
    reject_reason VARCHAR(200) NULL COMMENT '拒绝原因',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (id),
    UNIQUE KEY uk_strategy_id (strategy_id),
    KEY idx_command_id (command_id),
    KEY idx_status (status),
    KEY idx_risk_level (risk_level)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='策略表';

-- 人群画像表
CREATE TABLE smartad_crowd (
    id BIGINT(20) NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    crowd_tag VARCHAR(50) NOT NULL COMMENT '人群标签',
    user_count BIGINT(20) NOT NULL DEFAULT 0 COMMENT '预估人群数量',
    prefer_tags VARCHAR(200) NULL COMMENT '偏好标签，JSON数组格式',
    active_time VARCHAR(30) NULL COMMENT '活跃时段',
    age_range VARCHAR(20) NULL COMMENT '年龄范围',
    gender VARCHAR(10) NULL COMMENT '性别',
    city_level VARCHAR(20) NULL COMMENT '城市级别',
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (id),
    UNIQUE KEY uk_crowd_tag (crowd_tag)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='人群画像表';

-- 报表表
CREATE TABLE smartad_report (
    id BIGINT(20) NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    strategy_id VARCHAR(30) NOT NULL COMMENT '关联策略ID',
    report_date DATE NOT NULL COMMENT '报表日期',
    expose BIGINT(20) NOT NULL DEFAULT 0 COMMENT '曝光量',
    click BIGINT(20) NOT NULL DEFAULT 0 COMMENT '点击量',
    click_rate DECIMAL(5,4) NOT NULL DEFAULT 0 COMMENT '点击率',
    convert BIGINT(20) NOT NULL DEFAULT 0 COMMENT '转化量',
    convert_rate DECIMAL(5,4) NOT NULL DEFAULT 0 COMMENT '转化率',
    cost DECIMAL(10,2) NOT NULL DEFAULT 0 COMMENT '消耗金额',
    roi DECIMAL(5,2) NOT NULL DEFAULT 0 COMMENT '投入产出比',
    budget_deviation DECIMAL(5,4) NOT NULL DEFAULT 0 COMMENT '预算消耗偏差',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    PRIMARY KEY (id),
    UNIQUE KEY uk_strategy_date (strategy_id, report_date),
    KEY idx_report_date (report_date)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='报表表';

-- 告警表
CREATE TABLE smartad_alert (
    id BIGINT(20) NOT NULL AUTO_INCREMENT COMMENT '自增主键',
    alert_id VARCHAR(30) NOT NULL COMMENT '告警ID（格式：ALT+日期+序号）',
    alert_content VARCHAR(500) NOT NULL COMMENT '告警内容',
    alert_type VARCHAR(30) NOT NULL COMMENT '告警类型：接口异常/策略执行失败',
    related_id VARCHAR(30) NULL COMMENT '关联ID（策略ID或指令ID）',
    status VARCHAR(10) NOT NULL DEFAULT '未读' COMMENT '状态：未读/已读',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    read_time DATETIME NULL COMMENT '阅读时间',
    PRIMARY KEY (id),
    UNIQUE KEY uk_alert_id (alert_id),
    KEY idx_status (status),
    KEY idx_create_time (create_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='告警表';
```

## 字段类型说明

| 类型          | 使用场景                      |
| ------------- | ----------------------------- |
| BIGINT        | 主键ID、数量统计（曝光/点击） |
| VARCHAR       | 字符串标识、名称、JSON数组    |
| DECIMAL(10,2) | 金额（预算/消耗/出价）        |
| DECIMAL(5,2)  | ROI                           |
| DECIMAL(5,4)  | 比率（点击率/转化率/偏差）    |
| DATETIME      | 时间戳                        |
| DATE          | 报表日期                      |
| TINYINT       | 状态开关                      |
